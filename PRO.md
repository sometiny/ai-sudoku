# Sudoku 多人对战游戏 - 产品需求

## 项目概述

基于 Vue 3 + Node.js 的实时多人在线数独对战游戏。玩家可以创建或加入房间，与好友或其他玩家进行数独竞速比赛。

## 功能需求

### 1. 玩家系统

- 玩家自定义昵称
- 昵称保存在 localStorage，下次访问自动读取
- 无需注册登录，进入游戏即可设置昵称
- 支持修改昵称

### 2. 房间系统

| 功能 | 描述 |
|------|------|
| 创建房间 | 任意玩家可创建房间，设置房间名称和难度 |
| 加入房间 | 任意玩家可加入已存在的房间 |
| 离开房间 | 玩家可随时离开房间 |
| 房间列表 | 大厅显示所有等待中的房间 |
| 房主权限 | 创建者自动成为房主，可发起开始游戏 |
| 房主转移 | 房主离开时自动转移给其他玩家 |
| 人数限制 | 房间人数无上限 |

#### 房间状态

- `waiting` - 等待中，可加入
- `ready` - 准备开始，等待玩家确认
- `playing` - 游戏进行中
- `finished` - 游戏已结束

### 3. 开始机制

```
流程：
1. 房主点击"开始游戏"按钮
2. 系统检查参与玩家数量 ≥ 2
3. 广播"准备开始"消息，进入准备状态
4. 启动 10 秒倒计时
5. 所有参与玩家需在 10 秒内点击"同意"按钮
6. 10 秒倒计时结束：
   - 未点击"同意"的玩家 → 被踢出房间
   - 剩余玩家 ≥ 2 → 游戏正式开始
   - 剩余玩家 < 2 → 取消开始，返回等待状态

注意事项：
- 旁观者不需要确认
- 房主被踢出后，房主权限转移给其他玩家
- 倒计时期间新玩家不能加入参与，只能旁观
```

### 4. 观战系统

- 支持旁观者角色
- 游戏进行中可进入房间旁观
- 旁观者可见：
  - 实时棋盘状态（所有玩家）
  - 实时排行榜
  - 计时器
- 旁观者不可参与游戏操作
- 旁观者可随时离开

### 5. 对战模式

**竞速模式**

- 所有参与玩家获得同一道数独题目
- 同时开始计时
- 玩家独立填写自己的棋盘
- 实时排行榜显示：
  - 各玩家完成进度（已填格子数/总格子数）
  - 已完成玩家的用时排名
- 最先正确完成者获胜

### 6. 游戏规则

**数独规则**

- 标准 9×9 网格
- 每行包含 1-9 各一个
- 每列包含 1-9 各一个
- 每个 3×3 宫格包含 1-9 各一个

**游戏限制**

| 限制项 | 规则 |
|--------|------|
| 时间限制 | 5 分钟，超时自动结束游戏 |
| 完成判定 | 所有格子正确填写 |
| 结算 | 所有玩家完成或超时后显示最终排名 |

### 7. 难度设置

| 难度 | 空格数 | 描述 |
|------|--------|------|
| 简单 | 35 | 适合新手 |
| 中等 | 45 | 标准难度 |
| 困难 | 55 | 高难度挑战 |

### 8. 游戏交互

**基础操作**

- 点击格子选中
- 键盘数字键或点击数字面板输入
- 点击删除键清除数字

**辅助功能**

- 错误提示：填写错误数字时显示红色标记
- 笔记模式：小字标注候选数字（可选实现）

### 9. 游戏结算

- 显示所有玩家排名
- 显示各玩家用时
- 提供"再来一局"和"返回大厅"选项
- 房间可选择重新开始（房主权限）

## 非功能需求

### 性能要求

- 实时通信延迟 < 100ms
- 支持断线重连
- 页面加载时间 < 3s

### 兼容性

- 支持 Chrome、Firefox、Safari、Edge 最新版本
- 响应式设计，支持移动端访问

### 可维护性

- TypeScript 类型覆盖
- 模块化代码结构
- 清晰的代码注释

## 用户界面

### 页面流程

```
进入游戏 → 输入昵称 → 进入大厅
    ↓
大厅页面（房间列表）
    ↓
创建房间 或 加入房间
    ↓
房间等待页（等待玩家、准备开始）
    ↓
游戏页面（数独对战）
    ↓
结算页面（显示排名）
    ↓
返回大厅 或 再来一局
```

### 主要页面

1. **昵称设置页** - 首次进入，设置昵称
2. **大厅页面** - 房间列表、创建房间入口
3. **房间等待页** - 玩家列表、准备确认、旁观者列表
4. **游戏页面** - 数独棋盘、计时器、排行榜
5. **结算页面** - 最终排名、操作选项
